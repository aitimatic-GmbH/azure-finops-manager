# Security Policy

## Reporting Vulnerabilities

If you discover a security vulnerability in this repository, please report it via one of the following private channels:

- **GitHub Security Advisory (preferred):** Use GitHub's "Security" tab to privately report vulnerabilities.
- **Email:** `tec-cops@aitimatic.de`

**Please DO NOT open public issues for security vulnerabilities.**

## What to Include

To help us quickly analyze the issue, please include the following information in your report:
- A description of the vulnerability.
- The steps to reproduce the issue.
- An assessment of the potential impact.
- A suggested fix (if you have one).

## Our Commitment

- We strive to acknowledge receipt of your report **in a timely manner**.
- We will review every report carefully and address it according to our internal processes.

## Scope

This security policy applies to:
- The code in this repository.
- The configured GitHub Actions workflows.
- The directly included dependencies and submodules.

## Out of Scope

The following are not covered by this policy:
- Vulnerabilities in third-party repositories (please report them to the respective maintainers).
- General questions or feature requests that are not security-related.
